Client Report

Project Information

• Project Name: German Gear OR GearUpGerman (something to do with German)
• Project Logo: Something to do with Germany and cars?
• Client: Jake Doliskani
• Client Affiliation: McMaster Professor in the Computer Science Department

Project Data

• Create a functioning marketplace for users to trade German vehicles with one another (similar to Facebook Marketplace and AutoTrader)
• Build a simple front end which users can easily navigate to create vehicle listings and search/filter for vehicle listings
• Implement an easy to query database to store/manage vehicle data (i.e: make, model, year, price, etc)
• Database will support adding new listings, updating existing ones, and removing old or sold listings
• Ensure that user data is also safely stored and maintained in our database (i.e: password, email, username, phone-number, etc)

Project Audeince

• Target Users: Car lovers, especially those interested in German cars
• Niche: German vehicle market
• Device: More likely to see higher website traffic from desktop as dealing with cars is seen as a big financial decision, so users will feel safer/more confident using a larger screen to properly make/view listings. HOWEVER, a mobile-friendly interface is still necessary as people might casually scroll through and view the site on the go in order to keep an eye on certain listings, as well as new listings

Project Static Information

• Company name and logo will be present throughout the site, as well as a support email for any inqueries/issues
• Privacy policy explaining how everything that happens in person during a trade does not put us at fault (i.e: we are involved in providing a means of transaction, we simply are here to allow users to make listings)
• A FAQ page with contact information, and maybe an About Us page
• Some form of legal agreement/outline for users to view if they make a public listing with their account
• Bonus Features: Language options and a dark mode option for the web page

Project Design Elements

• Clean Layout: Minimalistic with a simple colour scheme so important vehicle information is ALWAYS easy to notice (i.e: functionallity priortized over unnecessary beauty)
• Intuitive Navigation: Have a sticky header with clear menu options, a search bar, dropdown menus to help filter searches, breadcrumbs for easier navigation
• Responsive Design: Web app should be equally functional for both desktop and mobile users
• Big Buttons: There should be buttons for people to create listings, contact sellers, delete listings, favourite a listing, etc.
• Zoom In Feature: Users should be able to zoom in to get a closer look of images corresponding to vehicle listings
• Messaging System: Hopefully an on-site personal DM messaging system between buyers and sellers can be set up (however; if it is not possible, we can just have the buyer's contact info attached to the listing)

Project Interactive Elements

• Price Comparison Tool: Enable users to compare prices of similar products from different sellers
• Favourites Tool: Allow users to save vehicle listings to look at for later
• Interactive Map: A map with pins for sellers in the user’s area
• Chatbot: A clickable chatbot that provies answers to some more niche questions that the FAQ might not have answers to
